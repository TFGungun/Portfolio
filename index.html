Nix

Nix : Ashes of The Phoenix

Highlights of my work in the development of Nix : Ashes of The Phoenix

Game Architecture

Spell System

The Spell System is architected to facilitate large variants of Spells. Spells can have different Aim Method, different Execution, Levels, Damage, and other Parameters. To satisfy these requirements, every spell is comprised of multiple small components that define how it is aimed, executed, how it progresses through level, how it reads parameters, and more.

Game Architecture

Enemy System

Unique enemies are what makes ARPG great, and Nix have lots of them. Every enemy, be it mobs or bosses are made possible with the Enemy System, where each of them is created by attaching multiple small components, defining the enemy's basic behaviour (e.g moving, attacking) combined with powerful State Machine System. Walking, Slithering, Jumping, Flying, the system was developed to make all of them possible.

Game Architecture

Quest System

Quest System works together with the Dialogue System to immerse players in the hilly-billy of Pluma's Edge. In Nix, different types of quest are available : Extermination, Collecting Drops, Unlocking Areas, Triggering Flag. The system is architected to ease the team's effort in creating new quests but still provide the flexibility of mix and matching quest objectives to create immersive quests.

UI/UX

Interactive UI

To further immerse players into the game, unique interfaces are created with different themes and interactions. In order to realize the Artist's design, I developed custom Unity UI and EventSystem components. In this example, we use a 3rd-party plugin called FancyScrollView and combined with custom EventSystem components to support Mouse, Keyboard, and Gamepad altogether.

Performance

Optimization

Through multiple methods of optimization (Animation System tailoring, Object Pooling, Audio Optimization, etc), performance rose from 9FPS to 25FPS in an extreme case of 1000 enemies on-screen.

Audio

FMOD

To create immersive audio, I utilized FMOD with its Unity Plugin. With FMOD, we succeeded in creating 3D Sounds, Immersive Ambience, and Seamless BGMs.

Tools

Tools Programming

In the process of developing Nix, there are multiple occassion where large hauling are needed. Editor Tools were developed to ease this process, such as hauling our Animation System.

Live-Ops

Steam Services

To complete players' experience, I connected Nix with Steam Achievements and Steam Cloud. This process creates better experience for the Achievers and ease players' process in changing device.

ICANDO

Highlights of my work in the development of ICANDO

Gameplay Mechanics

Drag Drop Swap

This mechanic was developed for gameplays where arranging is the core mechanic. In this example, player has to arrange letters to create a word that matches the picture.

Gameplay Mechanics

Path Tracing

Path-Tracing mechanic is developed for gameplays where player has to trace a set of nodes to create a path. In this example, player has to harvest the cactuses by moving the tractor to paths that touch the cactuses. Player can also create back and forth path.

Gameplay Mechanics

D-Pad Mechanic

For gameplays that needed 4-directional movement, this D-Pad can be utilized to control avatar's movement. In this example, player can control the lion so that it reaches its food (Sheep).

Gameplay Mechanics

Stringed Instrument

One unique gameplay that I created in ICANDO where player can hold and drag a cello's bow to create instrumental sounds. Players can also record their playing and play it again in the play session.

Gameplay Mechanics

Car Pool

Similiar to Pool, Players can drag a car back and release it, to launch their car into other lettered cars. To finish the stage, player has to hit the correct car to fill in the blank.

Gameplay Mechanics

Puzzle

To extinguish the fire, Players can arrange differently-shaped docks in order to create a path for the Avatar. Wrong path will force the player to restart and think of another path.

UI/UX & REST API

Toy UI

I also participated in creating User Interfaces and using REST APIs to fill these interfaces with data. One example is this toy cupboard UI where every toy is retrieved from the game database and player's database.

Tools

Tools Programming

Math Titans

Math Titans

Highlights of my work in the development of Math Titans

Game Architecture

Battle System

Math Titans' battle system is Turn-Based Battle where in each turn, each team will attack the other. In Player's turn, players are able to arrange a multiplication sentence with the available numbers and shoot projectiles according to the multiplication. Players' objective is to clear the enemies before they are defeated.

Game Architecture

Equipment System

To add progression for players, players can wear different equipments that increase their stats and change the numbers they can use in battle. The equipments also have unique look so players can mix & match to create their preferred appearance.

Game Architecture

Shop System

Coins acquired from battles can be spent in the Shop to buy new equipments. These equipments' prices varies according to their rarity and power.

Game Architecture

Map System

Players can progress further through the game by entering different battles from the map. Players can also see what equipments will be dropped from completing the levels.
